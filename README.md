طريقة الاستخدام

1. المتطلبات

· Python 3.6+
· لا حاجة لمكتبات خارجية (كل شيء يعمل بالمكتبات القياسية)

2. الأمثلة

مسح شبكة محلية باستخدام ICMP (ping):

```bash
python3 network_scanner.py -n 192.168.1.0/24
```

مسح باستخدام TCP على المنافذ 80 و443 فقط:

```bash
python3 network_scanner.py -n 10.0.0.0/24 -m tcp -p 80,443
```

مسح مع 100 خيط وحفظ النتائج في ملف result.txt:

```bash
python3 network_scanner.py -n 192.168.1.0/24 -t 100 -o result.txt -v
```

3. شرح الخيارات

الخيار الوظيفة
-n, --network النطاق بصيغة CIDR (مطلوب)
-m, --method طريقة المسح: icmp (ping) أو tcp (اتصال منفذ)
-p, --ports قائمة منافذ TCP مفصولة بفواصل (مثال: 22,80,443)
-t, --threads عدد الخيوط المتزامنة (يزيد السرعة)
-to, --timeout مهلة الانتظار لكل محاولة (ثواني)
-o, --output اسم ملف الإخراج (افتراضي: active_hosts.txt)
-v, --verbose طباعة كل IP نشط فور اكتشافه
--no-banner إخفاء الشعار الافتتاحي

4. مثال على ملف الإخراج

```
# نتائج مسح الشبكة
# التاريخ: 2025-03-29 14:35:12
# إجمالي الأجهزة النشطة: 5

192.168.1.1
192.168.1.10
192.168.1.22
192.168.1.100
192.168.1.254
```

---

التحسينات التي أضفتها

1. شريط تقدم ديناميكي – يعرض نسبة الإنجاز أثناء المسح.
2. دعم متعدد الخيوط – مسح سريع لمئات العناوين.
3. مرونة في طرق الكشف:
   · ICMP (ping) – سريع ومناسب للشبكات الداخلية.
   · TCP connect – يتحقق من فتح منافذ محددة (مفيد إذا كان ping محظوراً).
4. معالجة أنظمة التشغيل المختلفة – ping يعمل على Windows وLinux/macOS.
5. توقيت المسح – عرض الوقت المستغرق.
6. حفظ النتائج بصيغة منظمة مع تاريخ وعدد الأجهزة.
7. خيار verbose لعرض النتائج فور اكتشافها.

---

ملاحظات مهمة

· الدقة: ICMP قد لا يكتشف بعض الأجهزة التي تمنع ping. استخدم -m tcp -p 80,443 في هذه الحالة.
· السرعة: رفع عدد الخيوط (-t 100) يسرع المسح لكن قد يسبب ضغطاً على الشبكة.
· الاستخدام القانوني: استخدم هذه الأداة فقط على الشبكات التي تملك الإذن باختبارها.
